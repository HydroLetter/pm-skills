# 产品架构案例库

本skill汇集各行业真实产品架构案例，为产品设计提供参考。

## 一、电商零售类

### 1. 电商平台总体架构

**核心模块**：
```
前端层：
  ├─ 用户端：Web/App/小程序/H5
  ├─ 商家端：商家后台/App
  └─ 运营端：CMS/营销/数据

业务中台：
  ├─ 商品中心：类目/SPU/SKU/属性
  ├─ 交易中心：购物车/订单/履约
  ├─ 支付中心：多渠道支付/对账
  ├─ 库存中心：多仓库存/库存分配
  ├─ 营销中心：优惠券/秒杀/拼团
  └─ 用户中心：会员/积分/等级

基础服务：
  ├─ 搜索服务：商品搜索/推荐
  ├─ 消息服务：短信/推送/站内信
  └─ 文件服务：图片/视频存储
```

### 2. 供应链系统架构

**功能域划分**：

| 域 | 核心功能 | 系统 |
|----|----------|------|
| **采购域** | 供应商管理/采购订单/到货质检 | SRM |
| **库存域** | 入库/出库/盘点/调拨/库存分配 | WMS |
| **配送域** | 运力管理/配送调度/轨迹跟踪 | TMS |
| **财务域** | 结算/对账/发票/成本核算 | FMS |

**系统集成**：
- 上游对接：电商平台/线下门店
- 下游对接：第三方物流/仓储服务商

### 3. 无人便利店架构

**智能硬件层**：
- 门禁系统：扫码/人脸/NFC开门
- 监控系统：视频行为分析/防盗
- 支付系统：RFID识别/自动结算
- 温控系统：智能地暖/空调

**软件系统层**：
- 门店管理系统：商品/库存/价格
- 订单系统：线上线下订单统一
- 会员系统：积分/优惠券/储值
- 运维系统：设备监控/故障告警

### 4. 有赞生态架构

**产品矩阵**：
- 有赞微商城：小程序电商
- 有赞零售：门店数字化
- 有赞美业：预约/服务/会员
- 有赞餐饮：点餐/外卖/会员

**服务层**：
- 基础服务：店铺/商品/订单/支付
- 营销服务：优惠券/拼团/分销
- 数据服务：报表/分析/洞察

## 二、金融支付类

### 5. 支付平台架构

**核心能力**：
```
接入层：支付网关/API开放/鉴权验签
    ↓
产品层：快捷支付/网银支付/扫码支付
    ↓
渠道层：微信/支付宝/银联/银行直连
    ↓
核心层：账户系统/清结算/风控
```

**关键系统**：
| 系统 | 功能 |
|------|------|
| 收银台 | 统一支付入口，多支付方式聚合 |
| 账户系统 | 余额/积分/红包/储值卡管理 |
| 清结算 | 清分/结算/对账/差错处理 |
| 风控系统 | 反欺诈/交易监控/规则引擎 |

### 6. 银行系统架构

**渠道层**：
- 个人网银/企业网银
- 手机银行/微信银行
- ATM/POS/自助终端

**业务层**：
- 存款业务：活期/定期/理财
- 贷款业务：个人贷/企业贷/房贷
- 支付业务：转账/汇款/代扣
- 中间业务：代理/托管/咨询

**核心层**：
- 账户核心：总账/分户账
- 支付核心：支付指令/清算指令
- 产品工厂：灵活配置金融产品

### 7. 汽车金融平台架构

**业务全流程**：
```
贷前：客户申请 → 征信查询 → 授信审批
  ↓
贷中：合同签署 → 放款 → 车辆抵押登记
  ↓
贷后：还款管理 → 逾期催收 → 结清解押
```

**核心模块**：
- 审批系统：工作流引擎/规则引擎
- 风控系统：征信/反欺诈/评分卡
- 资金系统：放款/还款/对账

### 8. 现金贷/消费金融架构

**用户端**：
- 注册认证：实名/人脸识别/银行卡
- 借款申请：额度申请/期限选择
- 还款管理：主动还款/自动扣款

**管理端**：
- 客户管理：黑白灰名单/分级管理
- 产品管理：利率/期限/额度配置
- 风控管理：规则引擎/模型/预警
- 催收管理：催收策略/外呼/上门

## 三、CRM与客户服务类

### 9. CRM产品架构

**功能模块**：
```
营销获客：官网/名片/素材/活动
    ↓
销售管理：线索/商机/客户/订单
    ↓
客户服务：工单/知识库/呼叫中心
    ↓
数据分析：销售报表/客户分析
```

**客户模型**：
- RFM模型：最近消费/消费频次/金额
- 客户画像：标签/行为/偏好
- 生命周期：潜客/新客/活跃/流失

### 10. 客服平台架构

**服务渠道**：
- 在线客服：Web/APP/H5嵌入
- 呼叫中心：语音/IVR/录音
- 智能客服：AI机器人/知识库

**核心功能**：
- 工单系统：创建/分配/流转/闭环
- 知识库：FAQ/文档/视频教程
- 质检系统：录音抽检/评分/培训

## 四、AI与智能类

### 11. AI医疗产品架构

**应用场景**：
- 医学影像AI：CT/MRI/病理分析
- 辅助诊断：症状分析/用药推荐
- 智能问诊：NLP对话/导诊分诊

**技术架构**：
```
应用层：诊断/问诊/影像/慢病管理
    ↓
平台层：医学知识库/推理引擎/模型管理
    ↓
数据层：电子病历/医学文献/影像数据
    ↓
设备层：影像设备/可穿戴设备
```

### 12. 人脸识别产品架构

**功能模块**：
- 人脸检测：定位人脸位置
- 特征提取：提取128维特征向量
- 人脸比对：1:1验证/1:N搜索
- 活体检测：防照片/视频攻击

**应用场景**：
- 门禁考勤：刷脸通行
- 身份核验：金融开户/考试认证
- 安防监控：布控/轨迹追踪

### 13. AR产品架构

**技术层**：
- 图像识别：特征点检测/匹配
- 空间定位：SLAM/平面检测
- 渲染引擎：3D模型/特效叠加

**应用层**：
- AR营销：红包/优惠券/游戏
- AR导航：室内/室外导航
- AR教育：3D模型展示

### 14. 物联网中台架构

**设备接入层**：
- 协议适配：MQTT/CoAP/HTTP
- 设备管理：注册/认证/生命周期
- 消息路由：设备↔云端通信

**平台能力层**：
- 规则引擎：设备联动/告警触发
- 数据分析：时序数据/设备画像
- 远程控制：OTA升级/指令下发

## 八、架构学习建议

1. **先理解业务**：架构服务于业务，先懂业务再谈架构
2. **多看优秀案例**：学习大厂架构设计思路
3. **掌握分层思维**：表现层/应用层/服务层/数据层
4. **关注非功能需求**：性能、安全、扩展性、可用性
5. **持续迭代**：架构不是一次性设计，需随业务发展演进
