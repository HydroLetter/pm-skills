## 一、万能通用模型

### 1. SMART原则 — 目标设定

| 字母 | 含义 | 说明 | 反面例子 |
|------|------|------|----------|
| **S** | Specific 具体 | 清晰明确，不模糊 | "提升用户体验" ❌ |
| **M** | Measurable 可衡量 | 有数据指标 | "让用户更满意" ❌ |
| **A** | Attainable 可达成 | 跳一跳够得着 | "DAU翻10倍" ❌ |
| **R** | Relevant 相关 | 与整体目标一致 | 做与KPI无关的事 ❌ |
| **T** | Time-bound 有时限 | 有deadline | "尽快完成" ❌ |

**✅ 正确示例**：
> "在Q3结束前，将App次日留存率从35%提升至42%"

---

### 2. MECE原则 — 结构化分析

> **Mutually Exclusive, Collectively Exhaustive**
> 相互独立，完全穷尽 — 确保分析不遗漏、不重叠

**常用分解方法**：

| 方法 | 说明 | 示例 |
|------|------|------|
| **二分法** | 非A即B | 新用户/老用户、付费/免费 |
| **流程法** | 按流程步骤 | 获客→激活→留存→变现→推荐 |
| **要素法** | 按组成要素 | GMV = 流量 × 转化率 × 客单价 |
| **矩阵法** | 两维度交叉 | 重要-紧急四象限 |

**使用场景**：
- 拆解问题找到根因
- 分类整理信息
- 确保方案覆盖全面

---

### 3. 5W2H分析 — 落地执行

| 问题 | 核心 | 示例 |
|------|------|------|
| **What** | 是什么？做什么？ | 要上线会员体系功能 |
| **Why** | 为什么做？ | 提升用户LTV，增加收入 |
| **Who** | 谁来做？ | 产品A负责，研发B支持 |
| **When** | 什么时间做？ | Q2启动，6月底上线 |
| **Where** | 在哪里做？ | App端先上，Web端二期 |
| **How** | 怎么做？ | 分3阶段：基础版→进阶版→权益商城 |
| **How much** | 做到什么程度？ | 付费转化率提升至5% |

**使用场景**：
- 完善用户故事描述
- 项目启动前全面梳理
- 弥补考虑问题的疏漏

## 二、市场分析模型

### 4. PEST分析 — 外部宏观环境

| 维度 | 关注内容 | 示例 |
|------|----------|------|
| **P-政治** | 政策法规、监管态度、政治稳定性 | 数据安全法对隐私产品的要求 |
| **E-经济** | 经济周期、消费能力、市场规模 | 下沉市场消费潜力 |
| **S-社会** | 人口结构、用户习惯、文化趋势 | Z世代社交偏好 |
| **T-技术** | 新技术趋势、技术成熟度 | AI大模型对产品的颠覆 |

**使用场景**：
- 进入新市场前的宏观评估
- 年度战略规划
- 行业研究报告撰写

---

### 5. 波士顿矩阵 — 产品组合管理

```
                高增长率
                    ↑
    问题产品        |        明星产品
    (选择性投资)    |        (加大投资)
    ————————————————+————————————————→ 高市场份额
    瘦狗产品        |        金牛产品
    (撤退/放弃)     |        (维持收割)
                    ↓
                低增长率
```

| 类型 | 特征 | 策略 |
|------|------|------|
| **明星产品** | 高增长+高份额 | 加大投入，扩大优势 |
| **金牛产品** | 低增长+高份额 | 维持现状，收割利润 |
| **问题产品** | 高增长+低份额 | 评估潜力，选择性投入 |
| **瘦狗产品** | 低增长+低份额 | 考虑退出或止损 |

**使用场景**：
- 多产品线的资源分配
- 产品生命周期管理
- 投资组合决策

### 6. SWOT分析 — 产品发展战略

| | 有利因素 | 不利因素 |
|------|----------|----------|
| **内部** | **S-优势**<br>我们的强项 | **W-劣势**<br>我们的短板 |
| **外部** | **O-机会**<br>外部环境利好 | **T-威胁**<br>外部挑战 |

**战略组合**：

| 战略 | 组合 | 说明 |
|------|------|------|
| **SO战略** | 优势+机会 | 最大限度发展，进攻型 |
| **WO战略** | 劣势+机会 | 利用机会弥补劣势 |
| **ST战略** | 优势+威胁 | 利用优势抵御威胁 |
| **WT战略** | 劣势+威胁 | 收缩防守，规避风险 |

**使用场景**：
- 竞品分析
- 产品战略规划
- 核心功能方向制定

---

### 7. 波特五力 — 行业竞争态势

```
            供应商议价能力
                   ↑
    潜在进入者 ←  行业内竞争  → 购买者议价能力
    威胁         激烈程度       威胁
                   ↓
            替代品威胁
```

| 力量 | 评估问题 | 对行业利润的影响 |
|------|----------|------------------|
| **供应商议价能力** | 供应商集中度高吗？转换成本高吗？ | 高议价 → 利润↓ |
| **购买者议价能力** | 客户集中吗？信息透明吗？ | 高议价 → 利润↓ |
| **行业内竞争** | 竞争者多吗？增长快吗？ | 竞争激烈 → 利润↓ |
| **潜在进入者** | 进入壁垒高吗？ | 壁垒低 → 利润↓ |
| **替代品威胁** | 替代品好用吗？转换成本低吗？ | 威胁高 → 利润↓ |

**使用场景**：
- 评估行业吸引力
- 制定竞争策略
- 判断市场进入时机

## 三、产品运营模型

### 8. 用户体验要素 — 产品体验五层

> 从抽象到具体，从战略到表现

| 层级 | 核心问题 | 产出物 |
|------|----------|--------|
| **战略层** | 用户需求是什么？产品定位是什么？ | 用户画像、产品定位 |
| **范围层** | 功能范围是什么？内容需求是什么？ | 功能列表、内容规划 |
| **结构层** | 信息如何组织？交互流程如何设计？ | 信息架构、交互流程 |
| **框架层** | 界面如何布局？导航如何设计？ | 线框图、导航设计 |
| **表现层** | 视觉如何呈现？ | UI设计稿、视觉规范 |

**使用场景**：
- 竞品体验分析
- 产品体验报告撰写
- 与设计团队沟通

### 9. 马斯洛需求理论 — 需求层次挖掘

```
    ↑ 自我实现    (价值观、创造力、道德)
    ↓
    ↑ 尊重需求    (自尊、信心、成就、被尊重)
    ↓
    ↑ 社交需求    (友情、爱情、归属感)
    ↓
    ↑ 安全需求    (人身安全、健康保障、资源)
    ↓
    ↑ 生理需求    (食物、水、睡眠、性)
```

| 需求层级 | 产品体现 | 示例 |
|----------|----------|------|
| **生理** | 基础功能可用 | 能登录、能下单 |
| **安全** | 数据安全、隐私保护 | 隐私政策、数据加密 |
| **社交** | 互动、分享、社群 | 评论、点赞、圈子 |
| **尊重** | 等级、认证、专属服务 | VIP标识、专属客服 |
| **自我实现** | 成就系统、个人品牌 | 创作者中心、勋章 |

**使用场景**：
- 用户需求深度挖掘
- 产品功能层次设计
- 用户激励机制设计

---

### 10. KANO模型 — 需求优先级排序

| 需求类型 | 用户反应 | 产品策略 |
|----------|----------|----------|
| **基本型** | 不做会极度不满，做了认为是应该的 | 必须满足，门槛型 |
| **期望型** | 做得越好越满意，线性关系 | 持续优化，竞争型 |
| **兴奋型** | 做了超出预期，不做也没意见 | 制造惊喜，差异化 |
| **无差异** | 做不做用户没感觉 | 不投入 |
| **反向型** | 做了反而引起不满 | 坚决避免 |

**判断方法**：
- 问用户：如果**有**这个功能，您觉得怎么样？
- 问用户：如果**没有**这个功能，您觉得怎么样？
- 交叉分析得出需求类型

### 11. 产品画布 — MVP设计

**核心原则**：
> NOT LIKE THIS: 先做轮子→再做车身→再做引擎→最后完整车（用户全程不满意）
> 
> LIKE THIS: 先做滑板→滑板车→自行车→摩托车→汽车（每个阶段都可用）

**MVP设计要点**：
1. **最小化**：砍掉一切非核心功能
2. **可行性**：必须是可运行的完整产品
3. **验证性**：每个版本都能验证一个假设

**使用场景**：
- 新产品冷启动
- 功能快速验证
- 资源受限时的取舍

### 12. AARRR模型 — 运营增长漏斗

```
    ┌──────────┐
    │ Acquisition 获取 │ ← 用户从哪里来？
    └────┬─────┘
    ┌────┴─────┐
    │ Activation  激活 │ ← 用户第一次体验如何？
    └────┬─────┘
    ┌────┴─────┐
    │ Retention   留存 │ ← 用户会回来吗？
    └────┬─────┘
    ┌────┴─────┐
    │ Revenue     收入 │ ← 用户愿意付费吗？
    └────┬─────┘
    ┌────┴─────┐
    │ Referral    传播 │ ← 用户会推荐给别人吗？
    └──────────┘
```

| 环节 | 核心指标 | 优化方向 |
|------|----------|----------|
| **获取** | 新增用户数、CAC | 渠道ROI、获客效率 |
| **激活** | 首次体验完成率 | 引导流程优化 |
| **留存** | 次日/7日/30日留存 | 核心价值传递 |
| **收入** | ARPU、付费率 | 变现效率 |
| **传播** | 分享率、K因子 | 裂变机制 |

**使用场景**：
- 增长策略制定
- 数据分析框架
- 产品健康度评估

## 四、个人成长模型

### 13. 四象限法则 — 时间管理

```
                重要
                  ↑
    第二象限        |        第一象限
    计划去做        |        立刻去做
    (战略投入)      |        (危机处理)
    ————————————————+————————————————→ 紧急
    第四象限        |        第三象限
    能不做就不做    |        授权别人做
    (时间浪费)      |        (事务性工作)
                  ↓
                不重要
```

**PM应用示例**：
| 象限 | 任务类型 | 例子 |
|------|----------|------|
| 第一象限 | 紧急且重要 | 线上BUG、老板紧急需求 |
| 第二象限 | 重要不紧急 | 产品规划、能力建设 |
| 第三象限 | 紧急不重要 | 某些会议、例行报表 |
| 第四象限 | 不重要不紧急 | 无目的刷手机、闲聊 |

**策略**：
- 把更多时间投入**第二象限**（重要不紧急），减少第一象限的危机

### 14. 金字塔原理 — 沟通表达

> 结论先行，以上统下，归类分组，逻辑递进

**结构**：
```
        中心思想 (结论)
       /    |    \
   分论点A  分论点B  分论点C
   /  \     /  \     /  \
论据a 论据a 论据a 论据a 论据a 论据a
```

**应用场景**：

| 场景 | 金字塔应用 |
|------|-----------|
| **工作汇报** | 先说结论→再说论据 |
| **PRD写作** | 先讲目标→再讲方案→最后细节 |
| **需求评审** | 先讲价值→再讲方案→再讲排期 |

**两种推理方式**：
- **演绎推理**：大前提→小前提→结论（适合说服）
- **归纳推理**：多个事实→共同结论（适合汇报）

### 15. KISS模型 — 总结复盘

| 维度 | 问题 | 示例 |
|------|------|------|
| **Keep** 保持 | 什么做得好，继续？ | 每日站会提高了沟通效率 |
| **Improve** 改进 | 什么不够好，优化？ | 需求评审时间过长 |
| **Start** 开始 | 什么没做，要新增？ | 引入自动化测试 |
| **Stop** 停止 | 什么在浪费资源，要停？ | 停止开发未确认的需求 |

**使用场景**：
- 项目复盘会议
- 迭代回顾
- 个人周总结
